01 Core Concept

The player has access to a vulnerable Flask application running in a container. They SSH in, investigate, and harden it. When they believe they've fixed enough issues, they open the judge dashboard and press "Launch Payload Barrage".

The judge fires a shuffled batch of payloads at the app — some are malicious (SQLi, XSS, path traversal, etc.) and some are legitimate (normal logins, valid form submissions, standard API calls). The player's app must block the bad and serve the good.

Key Mechanic

Every legitimate payload that succeeds → +1 point

Every malicious payload that succeeds → −1 point

Payloads are shuffled and interleaved. The player doesn't know which is which until results are revealed. This prevents "block everything" strategies and forces real understanding — you must let good traffic through while stopping the bad.

The player can press the button as many times as they want. Each press runs a fresh barrage and the score resets — only the latest run counts. This lets them fix iteratively: barrage → see what failed → fix → barrage again.

The judge never modifies the app container. It only sends HTTP requests and SSH connection attempts as an external attacker would. The docker socket mount is only used to check file permissions and installed packages inside the container via docker exec for config-level checks that can't be tested over the network.

03 Scoring Model

Point Calculation

Each barrage fires all payloads (malicious + legitimate) in random order. The final score for that run:

formula
score = (legitimate payloads passed) − (malicious payloads passed) + (hygiene checks passed)

# Example: 20 network payloads (10 legit, 10 malicious) + 5 hygiene checks
# App blocks 8 attacks but also accidentally blocks 2 legit requests.
# Player fixed 3 out of 5 config issues.
# legit_passed = 8, malicious_passed = 2, hygiene_passed = 3
# score = 8 − 2 + 3 = 9  →  unlocks flags 1, 2, and 3

Flag Thresholds

Threshold Flag Meaning
≥ 1 point FLAG{f1r5t_bl00d_d3f3nd3r} You blocked at least one attack more than you broke
≥ 3 points FLAG{g3tt1ng_th3_h4ng_0f_1t} Solid start — several attack vectors closed
≥ 6 points FLAG{bl03_t34m_v3t3r4n} Most attacks blocked, most legit traffic flowing
≥ 10 points FLAG{f0rtr3ss_m0d3_4ct1v4t3d} Near-perfect defense — app is hardened and functional
Negative Scores Possible
If the player over-hardens (e.g., blocks all traffic) or under-fixes (attacks still work), the score can go negative. A score of −5 means more attacks succeeded than legitimate requests — the app is worse than useless.

Why This Works

The mixed payload approach creates a natural tension: you can't just firewall everything because legitimate traffic must pass. You can't leave things open because attacks will subtract points. And because the payloads are shuffled, the app logs show a realistic mix — the player has to actually understand what malicious traffic looks like in the noise.

Hygiene Checks (+1 each if passed)

These run via docker exec before the network barrage and test container-level hardening that can't be observed over HTTP.

Maximum Possible Score
10 (all legit pass) − 0 (all attacks blocked) + 5 (all hygiene pass) = 15 points. This means even with a perfect run, the player unlocks all four flags. The thresholds are tuned so that partial fixes still earn progress.
Log Obfuscation
All 20 network payloads fire in random order with random jitter (0.1–0.5s between each). In the app's access logs, the legitimate and malicious requests are interleaved — the player can't easily distinguish them by watching logs in real time. This simulates realistic traffic where attacks hide in normal usage.

# ─────────── HYGIENE (docker exec) ───────────

def _exec(cmd):
    r = subprocess.run(["docker", "exec", "vulnerable-app"] + cmd,
                       capture_output=True, text=True, timeout=10)
    return r.stdout

def h01_rogue_uid0():
    passwd = _exec(["cat", "/etc/passwd"])
    uid0 = [l.split(":")[0] for l in passwd.splitlines()
            if len(l.split(":")) > 2 and l.split(":")[2] == "0"]
    clean = uid0 == ["root"]
    return {"id": "H-01", "name": "Rogue UID-0",
            "type": "hygiene", "succeeded": clean,
            "detail": "Only root" if clean else f"UID-0: {uid0}"}

def h02_malicious_cron():
    cron = _exec(["cat", "/etc/cron.d/system-update"])
    clean = "curl" not in cron and "wget" not in cron
    return {"id": "H-02", "name": "Malicious Cron",
            "type": "hygiene", "succeeded": clean,
            "detail": "Clean" if clean else "Suspicious cron"}

def h03_entrypoint_backdoor():
    entry = _exec(["cat", "/app/entrypoint.sh"])
    patterns = ["/dev/tcp", "bash -i", "nc -e", "mkfifo"]
    clean = not any(p in entry for p in patterns)
    return {"id": "H-03", "name": "Entrypoint Backdoor",
            "type": "hygiene", "succeeded": clean,
            "detail": "Clean" if clean else "Backdoor found"}

def h04_suid_find():
    perms = _exec(["stat", "-c", "%a", "/usr/bin/find"]).strip()
    clean = not perms.startswith("4")
    return {"id": "H-04", "name": "SUID find",
            "type": "hygiene", "succeeded": clean,
            "detail": "SUID removed" if clean else "SUID set"}

def h05_attack_tools():
    nmap = _exec(["which", "nmap"]).strip()
    nc = _exec(["which", "nc"]).strip()
    clean = not nmap and not nc
    return {"id": "H-05", "name": "Attack Tools",
            "type": "hygiene", "succeeded": clean,
            "detail": "Removed" if clean else "Still installed"}

barrage.py — Payload Engine

pythonimport random, time
from payloads import ALL_PAYLOADS, HYGIENE_CHECKS

FLAG_THRESHOLDS = [
    (1,  "FLAG{f1r5t_bl00d_d3f3nd3r}"),
    (3,  "FLAG{g3tt1ng_th3_h4ng_0f_1t}"),
    (6,  "FLAG{bl03_t34m_v3t3r4n}"),
    (10, "FLAG{f0rtr3ss_m0d3_4ct1v4t3d}"),
]

def run_barrage():
    # 1. Hygiene checks (docker exec)
    hygiene_results = []
    for check in HYGIENE_CHECKS:
        try:
            hygiene_results.append(check())
        except Exception as e:
            hygiene_results.append({
                "id": check.__name__, "name": check.__name__,
                "type": "hygiene", "succeeded": False,
                "detail": f"Error: {e}"})

    # 2. Shuffle and fire network payloads
    shuffled = ALL_PAYLOADS[:]
    random.shuffle(shuffled)

    payload_results = []
    for fn in shuffled:
        try:
            result = fn()
        except Exception as e:
            result = {"id": fn.__name__, "name": fn.__name__,
                      "type": "unknown", "succeeded": False,
                      "detail": f"Error: {e}"}
        payload_results.append(result)
        time.sleep(random.uniform(0.1, 0.5))  # jitter

    # 3. Score
    legit_ok = sum(1 for r in payload_results
                   if r["type"] == "legitimate" and r["succeeded"])
    mal_ok = sum(1 for r in payload_results
                 if r["type"] == "malicious" and r["succeeded"])
    hyg_ok = sum(1 for r in hygiene_results if r["succeeded"])

    score = legit_ok - mal_ok + hyg_ok
    flags = [f for t, f in FLAG_THRESHOLDS if score >= t]

    return {
        "score": score,
        "breakdown": {
            "legit_passed": legit_ok, "legit_total": 10,
            "malicious_passed": mal_ok, "malicious_total": 10,
            "hygiene_passed": hyg_ok, "hygiene_total": 5,
        },
        "flags": flags,
        "payload_results": payload_results,
        "hygiene_results": hygiene_results,
    }

app = Flask(__name__)

09 Dashboard UI Spec

Before any barrage: Welcome text with rules and a large "Launch Payload Barrage" button. No scores, no flags, no results.

During barrage (~10-15s): Live feed showing payloads firing with anonymized labels ("Payload 01", "Payload 02"…). No types revealed. A progress bar fills as payloads complete.

After barrage: A dramatic reveal sequence plays — each payload's type flips from ??? to MALICIOUS or LEGITIMATE. The score counts up. Flag thresholds are shown as a progress track with locks that animate open as thresholds are met. Unlocked flags display in full. A detailed results table shows every payload with its outcome.

Re-run: The button stays available. Each press clears previous results and runs a fresh barrage. Only the latest run matters.

UX Detail — The Reveal
The type reveal is the core dramatic moment. During the barrage, results stream in anonymized. After all 20 complete, a 3-second countdown begins. Then each row flips its type label one by one (150ms stagger), coloring red or green. Succeeded malicious payloads pulse red. The score counter animates from 0 to final value. This makes the "did my fix work?" moment visceral and memorable.

The Core Tension
The barrage includes legitimate traffic that must succeed. If you break normal functionality while fixing security issues — overly aggressive input filtering, disabling endpoints, killing services — those legitimate payloads will fail and cost you points. Real defense means blocking attacks without breaking the app. That's the skill being tested.
